# ArtBridge 시연 시나리오 · 스크립트 (v0.8 초안)

> 발표 라이브 시연용. 실제 동작하는 범위만 구성. **결제는 Mock(항상 성공)** 이므로 정직하게 안내한다.
> 라이브 사고 대비 **동일 흐름 3~4분 백업 영상**을 반드시 준비한다.

## 1. 라이브 시연 흐름 (약 3~4분)

> 핵심 메시지: **"팬이 작가를 발견 → 포트폴리오로 신뢰 → 신청·계약·결제까지 한 흐름으로 이어지는 양면형 거래"**

| # | 화면(경로) | 행동 | 발표 멘트(요지) | 비고 |
|---|---|---|---|---|
| 1 | 탐색 `/` · `/creators` | 카테고리 필터·검색으로 작가 탐색 | "팬은 분야별로 창작자를 발견합니다" | 공개 화면, 로그인 불필요 |
| 2 | **포트폴리오** `/creators/[id]` 소개탭 | 스탯행 → 작가소개 → **작품 갤러리** → 진행 프로그램 → 멤버십 CTA 스크롤 | "이번 통합에서 **류진님 포트폴리오 흐름을 한진님 실데이터 위에 이식**한 핵심 화면입니다" | ★우리 작업 강조 포인트 |
| 3 | 프로그램 `/programs/[id]` | (팬) 프로그램 상세 → 신청 | "관심 프로그램에 참여 신청" | 무료=즉시 수락 / 유료=결제 단계로 |
| 4 | 작가 대시보드 `/dashboard/creator` | (작가) 신청자 확인 → 수락 | "작가는 신청을 검토하고 수락" | 수락 시 팬에게 알림 생성 |
| 5 | 알림 `/notifications` | (팬) 수락 알림 확인 | "수락 알림이 실시간 인앱으로 전달" | 외부 푸시 없음(인앱 전용) |
| 6 | 계약 `/contracts/[id]` | 금액 조율 → **양측 서명** → 결제(Mock) | "계약 금액 합의 후 양측 서명, 에스크로 결제" | **결제 Mock 안내 필수** |
| 7 | 리뷰 | 완료 승인 후 **상호 리뷰**(팬↔작가) | "거래 완료 후 양방향 평가로 신뢰 누적" | 양방향 리뷰 |

> **시연 안정화 팁:** 4~7단계는 라이브로 전 과정을 새로 만들면 시간이 길고 사고 위험이 큽니다.
> 시드에 **이미 진행 중인 신청·계약 상태**가 있으므로, 신청(3)까지만 라이브로 보여주고
> 수락·계약·결제 화면(4~7)은 **미리 세팅된 상태를 열어 화면 위주로** 보여주는 구성을 권장합니다.

## 2. 백업 영상 가이드 (3~4분)

- 위 1~7 흐름을 **동일 순서**로 화면 녹화 (1080p, 마우스 강조 권장)
- 각 단계 2~3초 머무르며 핵심 화면이 보이도록
- 결제 단계에 "Mock 결제" 자막 1줄
- 파일: `artbridge-demo-backup.mp4` (로컬 + 드라이브 백업)

## 3. 발표 중 트러블슈팅

| 상황 | 대응 |
|---|---|
| 배포 사이트 느림/오류 | 즉시 **백업 영상**으로 전환 |
| 로그인 실패 | 미리 로그인해 둔 시크릿 창 사용 / 계정·비번 재확인(`demo1234!`) |
| 결제 관련 질문 | "데모는 Mock 결제(항상 성공). 실 PG(PortOne/Toss)는 키 연동 시 동작하도록 추상화돼 있음" |
| 데이터가 바뀜 | 공유 DB라 다른 계정 활동이 반영될 수 있음 — 시연 직전 상태 확인 |

## 4. 정직성 멘트 (구현현황 기준, 과장 금지)

- **결제**: Mock 결제로 시연(실 PG 미연결). 계약·작품주문 경로는 키 연동 시 실 PG 전환 가능하도록 설계.
- **Google 로그인**: 키 있을 때만 실제 OAuth, 없으면 비활성(가짜 로그인 아님). 데모는 이메일/비번 사용.
- **AI 추천**: 키 없으면 결정론적 Mock. **이미지 업로드**: 로컬 저장(서버리스 비영속).
