폴백 정책 요약 (과장 금지 핵심)

  • 결제: resolvePaymentProvider()는 계약·작품주문에서만 사용. 멤버십 가입·포스트 유료구매는 mockPaymentProvider를 코드상 직접 호출하므로 PG 키가 있어도 항상 Mock 결제. 실제 PG는 계약/작품주문 경로 + env 3종(STORE_ID/CHANNEL_KEY/API_SECRET) 모두 있을 때만.
  • AI: 키 없거나 호출 실패 시 suggestMock(djb2 시드 결정론적). 예외를 throw하지 않음.
  • 이미지: 외부 스토리지 미구현. 로컬 파일시스템 저장만 존재(데모 한정, 영속성 한계).
  • 소셜로그인: env 없으면 OAuth 미실행·안내 리다이렉트.
