# ArtBridge 라이브 시연 각본 (목표 3분 30초~4분)

> **흐름:** 발견 → 신뢰(포트폴리오) → 거래(상태머신·에스크로) → 신뢰 누적(리뷰). 모든 화면은 실제 DB·인증·트랜잭션으로 동작(목업 아님).
> **핵심 원칙:** ① 핵심 작업물인 **포트폴리오 '소개' 탭**에 시간을 가장 많이 배분 ② 되돌리기 어려운 액션(완료 승인하기·멤버 제외·계약 액션)은 **클릭하지 않고 가리키며 설명** ③ 정직성 멘트 필수 — **결제는 Mock, 상태 전이는 실제 DB 트랜잭션**이라는 구분을 명확히.
> URL: `https://cadet-project.vercel.app` · 공통 비번: `demo1234!` · 메인은 라이브, 문제 시 `demo-backup.mp4`.

## 막 2. 신뢰 — 포트폴리오 '소개' 탭 ★이번 통합의 핵심 작업물 (0:40 ~ 2:00 · 약 80초) · 창A

작가 상세 진입 시 기본 탭은 **소개**. 위→아래로 천천히 스크롤하며 각 섹션을 짚는다. **시간을 가장 많이 배분 — 줄이지 않는다.**

**🖱️ 동작**
- 상단 탭 라벨 가볍게 가리킴: **소개(기본) / 포스트 / 멤버십 / 작품 / 클럽 / 커뮤니티**
- '소개' 탭 스크롤 순서: ① 스탯행(작가 노트·후원자·프로그램 수) → ② 작가 소개 bio → ③ 평점 → ④ 작품 갤러리 → ⑤ 진행 중인 프로그램 → ⑥ "멤버십으로 더 가까이" + 멤버십 참여하기 버튼

**🗣️ 멘트 (스크롤하며)**
> "두 번째 단계는 **신뢰**이고, 이 작가 '소개' 탭이 **이번 통합 작업의 핵심 결과물**입니다. 두 팀의 결과물을 합치면서, 작가를 **한 번의 스크롤로 파악**하도록 포트폴리오형으로 재구성했습니다."
>
> "위에서부터 보시면 — **상단 통계**로 작가 노트·후원자·프로그램 수를 보여주고, **작가 소개와 평점**, **작품 갤러리**, **진행 중인 프로그램**, 그리고 하단 **멤버십**까지 이어집니다."
>
> "여기 **평점**은 광고가 아니라, 뒤에서 보실 거래와 **양방향 리뷰**가 실제로 쌓여 만들어지는 값입니다. 발견에서 시작한 관심이 이 한 화면에서 신뢰로 이어지도록 설계했습니다."

**💡 포인트**: 한 번의 스크롤로 작가 전체를 보여주는 게 핵심. 내부 병합 구도(한진 base/류진 포트폴리오 이식)는 **질문 시에만** 답변.

## 막 3. 거래 — 신청 상태머신 + 에스크로 (2:00 ~ 3:00 · 약 60초 · 압축 안전 밸브 있음) · 전부 창A

팬은 이미 신청·결제·참여확정까지 진행된 상태(시드 활용)를 보여준다. **이 막은 전부 창A에서 진행 — 창 전환 없음.**

**🖱️ 동작 — ① 신청 라이프사이클**
- 창A 네비 **내 신청**(`/dashboard/fan/applications`) 클릭
- 상단 **필터 탭**을 좌→우로 짚으며 읽기: **전체 / 신청됨 / 결제 대기 / 참여 확정 / 진행 / 완료 / 거절**

**🗣️ 멘트**
> "세 번째 단계는 **거래**입니다. '내 신청' 화면을 보면 신청 한 건의 전체 라이프사이클이 보입니다 — 신청됨 → 결제 대기 → 참여 확정 → 진행 → 완료, 거절 분기까지. 단순 폼 제출이 아니라 **상태머신**으로 동작하고, 각 전이가 실제 DB 트랜잭션으로 기록됩니다. 이 계정은 이미 결제까지 마친 '참여 확정' 상태입니다."

**🗣️ 정직성 멘트 (이 막에서 반드시)**
> "여기서 정확히 짚고 가겠습니다. 오늘 보시는 결제는 **데모용 Mock 결제**입니다(실 PG 미연결). 다만 **계약·작품주문 경로는 키만 연동하면 실제 PG(PortOne/Toss)로 전환**되도록 추상화돼 있고, 멤버십·포스트 결제는 데모 정책상 Mock으로 고정했습니다."

**🖱️ 동작 — ② 에스크로 (가리키며 설명, 클릭 금지)**
- 창A에서 참여확정한 **프로그램 상세(demo-program-1)** 로 이동
- **"완료 승인하기"** 버튼을 마우스로 **가리키기만** 한다 (절대 클릭 금지, 커서는 버튼에서 살짝 떨어뜨림)

**🗣️ 멘트 (가리키며, 누르지 않음)**
> "거래의 핵심은 신뢰를 지키는 장치입니다. 팬이 낸 돈은 작가에게 바로 가지 않고 **에스크로에 묶입니다.** 작업이 끝나면 이 **'완료 승인하기'** 로 에스크로가 풀려 작가에게 정산되는 구조입니다."
>
> "한 가지 정확히 말씀드리면 — 이 버튼은 실제 송금이 아니라 **에스크로 상태를 바꾸는 DB 트랜잭션**입니다. 결제 자체는 Mock이지만 상태 전이는 실제로 기록됩니다. 누르면 상태가 바뀌어 데모상 되돌리기 번거로워, 오늘은 **클릭 없이 설명만** 드립니다."

> ⏱️ **압축 안전 밸브 (시간 초과 조짐 시)**: ② 에스크로에서 프로그램 상세 이동을 생략하고 '내 신청' 화면에 머문 채 구두로만 설명 — "프로그램 상세에 '완료 승인하기'가 있고, 누르면 에스크로가 풀려 정산됩니다. 실제 송금이 아니라 에스크로 상태를 바꾸는 DB 트랜잭션입니다."

**💡 포인트**: '거래'를 단순 결제가 아니라 **에스크로로 보호되는 약속 + 상태머신**으로 프레이밍. '결제는 Mock / 상태 전이는 실제 DB' 구분을 같은 호흡에.

## 막 4. 신뢰 누적 + 클로징 (3:00 ~ 3:40 · 약 40초) · 창B로 전환 → 마무리

막3을 전부 창A에서 끝냈으므로, **막4 시작에서 창B(작가)로 전환**한다. 창B는 사전 세팅대로 **관리 홈**에 띄워둔 상태.

**🖱️ 동작**
- 창B로 전환 → **관리 홈**(`/dashboard/creator`) → **퀵메뉴** 짚기: 작품 관리 / 작업 관리 / 포스트 작성 / 내 프로그램 / 프로그램 만들기 / 멤버 관리 / 멤버십 관리 / 내 커뮤니티
- (선택) **신청자 관리** 화면 → 참여자 항목과 **'멤버 제외'** 버튼을 **가리키기만** (클릭 금지)
- 계약 화면(`/contracts/[id]`)은 **말로만** 언급 (이 팬 계정 권한/상태에선 액션 버튼이 안 보임)

**🗣️ 멘트**
> "마지막입니다. 이 모든 건 한쪽만의 화면이 아닙니다. 작가 입장으로 와보면 — 작가는 여기서 작품·프로그램·멤버·정산까지 직접 운영합니다. 신청자 관리 화면을 보면, 팬이 낸 신청이 작가 화면에 그대로, **같은 하나의 DB 위에서 양방향으로** 연결됩니다."
>
> "그리고 **리뷰는 거래 완료 후 양쪽이 서로 남기는 구조**로 설계돼 있고, 그 값이 막2에서 보신 **평점으로 누적**됩니다. 오늘은 화면 시연 대신 설명으로 드립니다."

**🗣️ 정직성 멘트 (마무리에 압축)**
> "정직하게 한 번 더 정리하면 — 결제는 Mock(계약·작품주문은 키 연동 시 실 PG 전환, 멤버십·포스트는 Mock 고정), 구글 로그인은 키 있을 때만 실제 OAuth·없으면 비활성이라 오늘은 이메일/비번, AI 추천은 Mock 폴백, 이미지 업로드는 서버리스 한계로 로컬 저장입니다."

**🗣️ 클로징**
> "정리하면 — **발견, 신뢰(포트폴리오), 거래(상태머신·에스크로), 신뢰 누적(양방향 리뷰)** 의 한 사이클이 전부 실제 DB·인증·트랜잭션 위에서 동작합니다. MVP 우선순위는 신뢰에 뒀고, 그래서 포트폴리오 통합에 가장 공을 들였습니다. 배포본 44개 페이지가 전부 정상 동작합니다. 이상입니다. 감사합니다."

## 정직성(과장 금지) 멘트 모음 — 질문/필요 시 인용

- **결제**: "데모는 Mock 결제로 시연합니다(실 PG 미연결). 계약·작품주문 경로는 키 연동 시 실 PG(PortOne/Toss)로 전환되도록 추상화, 멤버십·포스트 결제는 Mock 고정입니다."
- **에스크로/완료 승인하기**: "'완료 승인하기'는 실제 송금이 아니라 에스크로 상태를 바꾸는 DB 트랜잭션입니다. 결제 자체는 Mock이지만 상태 전이는 실제로 기록됩니다."
- **Google 로그인**: "키가 있을 때만 실제 OAuth로 작동하고, 없으면 비활성입니다. 가짜 로그인이 아니라 오늘은 이메일/비밀번호로 시연합니다."
- **AI 추천**: "키가 없으면 Mock 폴백으로 동작합니다."
- **이미지 업로드**: "로컬 저장 방식이라 서버리스 환경의 영속성 한계가 있습니다."
- **리뷰**: "양방향 리뷰는 거래 완료 후 양쪽이 남기는 구조이고, 그 값이 평점으로 누적됩니다. 오늘은 설명으로 드립니다."
- **계약 화면**: "이 팬 계정 권한/상태에선 액션 버튼이 안 보여, 계약·에스크로는 클릭 없이 설명 위주로 진행합니다."
- **전체**: "배포본 44개 페이지 라우트가 전부 정상 작동하고, 차단 오류는 0건입니다."
