# ArtBridge 종단간 체험 점검 — 개선 계획서

점검 방식: 크리에이터/팬 두 역할로 실제 브라우저(headless Chromium) 스크린샷 체험. PRD 18장 데모 17단계 + 엣지케이스. dev server `localhost:3000`(Supabase 연결, 시드 데이터, 데모 계정 `demo1234!`).

결론 요약: **핵심 데모 흐름은 대부분 동작**(멤버십 가입→잠금해제, 계약 서명→Mock 결제 수수료 10% 정확, 완료 처리, 리뷰 폼/자격, 접근제어, 중복/자기신청 차단, 빈 상태 처리, AI mock 폴백). 다만 **데모를 끊을 수 있는 P0 1건**과 동선/피드백/카피 개선점이 확인됨. 모든 항목은 6/21 Freeze 후 허용 범위(버그·링크·빈화면·카피·polish)에 해당.

## P0 — 데모 차단 가능 (최우선 수정)

### 1. 메시지 없이 프로그램 참여 신청 시 "Validation failed" 실패
- **현상**: 프로그램 상세에서 메시지를 비우고 "참여 신청" 클릭 → 빨간 "Validation failed" 에러, 신청 안 됨. 메시지를 입력하면 정상 신청됨(DB 확인).
- **재현**: 팬 로그인 → `/programs/[id]` → 메시지 공란 → 참여 신청
- **원인**: `src/lib/validation/application.ts:12` `message: z.string().max(1000).optional()` 은 `undefined`만 허용(`null` 거부). 반면 `src/components/programs/ApplyButton.tsx:53` 은 빈 메시지일 때 `{ message: message || null }` 로 **null 전송** → zod 검증 실패.
- **권장 조치(택1, 1줄)**:
  - ApplyButton에서 `message: message.trim() ? message : undefined` 로 전송, 또는
  - 스키마를 `message: z.string().max(1000).nullish()` (null 허용)로 변경
- **영향**: 데모 스크립트 11단계(팬 참여 신청)가 자연스러운 입력(메시지 생략)에서 깨짐.
- **Freeze**: 허용(버그 수정)

## P1 — 데모 체감 저하 / 혼란

### 2. 크리에이터가 로그인하면 팬 홈(/creators)으로 잘못 이동
- **현상**: 크리에이터 계정으로 폼/데모버튼 로그인 시 `/dashboard/creator`가 아니라 **팬 홈 `/creators`로 리다이렉트**(세션은 정상 생성). 데모 "크리에이터로 시작하기"도 동일.
- **원인**: `src/app/login/actions.ts` — `signIn(redirect:false)` 직후 같은 요청에서 `getCurrentUser()`가 방금 설정된 세션을 못 읽어 `role=undefined` → `roleHome(undefined)`가 FAN 홈(`/creators`)으로 fallback(18행). callbackUrl이 있으면(보호라우트 경유) 정상.
- **권장 조치**:
  - `loginAsDemo(role)`: 이미 role 인자를 알고 있으므로 `roleHome`을 role로 직접 계산
  - `loginWithCredentials`: 이메일로 사용자 role을 사전 조회하거나 `signIn`의 `redirectTo` 사용
- **영향**: 데모 2단계(크리에이터 로그인 → 대시보드)에서 한 번 더 클릭 필요.
- **Freeze**: 허용(버그 수정)

### 3. 크리에이터 "내 클럽" 목록에 신청 관리 진입 링크 없음
- **현상**: `/dashboard/creator/programs` 에 "편집/삭제"만 있고 **신청 관리(applications) 링크가 없음**. 데모 12단계(신청 수락)로 가려면 URL 직접 입력 필요.
- **원인**: 프로그램 목록 항목에 `/dashboard/creator/programs/[id]/applications` 링크 미배치(페이지 자체는 정상 동작).
- **권장 조치**: 프로그램 카드에 "신청 관리" 링크/버튼 추가.
- **Freeze**: 허용(링크 연결)

### 4. 주요 액션 후 UI 반영 지연 (4~10초)
- **현상**: 로그인/계약 서명/결제/멤버십 가입/신청 후 "처리 중…" 이 길게 노출되고 "신청 완료"·"가입 완료" 가 즉시 안 뜸(기능 자체는 성공). 결제 완료 화면 도달까지 서명 5초+결제 10초 관측.
- **원인**: Supabase 원격 왕복 + bcrypt + `router.refresh()` 서버 재렌더 지연. 랜딩 첫 응답도 2.6초.
- **권장 조치**: 낙관적 UI/명시적 로딩 인디케이터 보강, Supabase 연결 풀(pooler) 리전·세션 재사용 점검. (발표 환경에서 데이터 미리 워밍업)
- **Freeze**: 허용(빌드/환경 정리, UI polish)

### 5. 날짜 표시 Hydration mismatch
- **현상**: 신청관리·알림·커뮤니티 등에서 dev "Issues" 누적(콘솔 hydration 에러). 기능엔 영향 없으나 깜빡임 발생.
- **원인**: `new Date(x).toLocaleString("ko-KR")` 를 클라이언트 컴포넌트에서 사용 — 서버(UTC/호스트) vs 클라이언트(KST) 타임존 불일치. 위치: `src/components/applications/ApplicationList.tsx:125`, `src/components/notification/NotificationList.tsx:130`, `src/components/community/CommunityPostList.tsx:37` 등.
- **권장 조치**: 서버에서 포맷해 문자열로 전달, 또는 `timeZone: "Asia/Seoul"` 고정 + 해당 노드에 `suppressHydrationWarning`.
- **Freeze**: 허용(버그 수정)

## P2 — Polish / 일관성 (여유 시)

### 6. 비로그인 공개 페이지에 글로벌 헤더/네비 없음
- 랜딩·`/creators`·`/programs` 비로그인 상태에 상단 네비가 없어 로그인/페이지 이동 동선이 약함(헤더는 로그인 후에만 렌더). 비로그인용 최소 헤더(로고+로그인 버튼) 권장.

### 7. AI 추천 카피
- 키 없는 데모/배포에서 "AI 일시적 오류로 기본 추천을 표시합니다" 가 **항상** 노출(부정적). "AI 추천(데모)" 같은 중립 문구 권장.
- 주차 구성 텍스트가 "1주차 — 1주차: …" 로 라벨 중복. mock 생성 포맷 정리.

### 8. 팬 대시보드 신청 상태 영어 ENUM 노출
- `/dashboard/fan` "최근 신청" 이 `ACCEPTED`/`PENDING` 영문 그대로. 신청관리 페이지는 "수락"/"대기 중" 한글 — 한글 매핑으로 통일.

### 9. 역할 가드 응답 불일치
- 로그인한 팬이 크리에이터 하위 페이지(`/dashboard/creator/programs/new`, `/.../applications`)에 접근하면 **`/login`으로** 보냄(이미 로그인했는데 재로그인 유도). `/dashboard/creator` 최상위는 `/creators`로. 가드 동작을 역할 홈 리다이렉트로 통일 권장.

### 10. 스튜디오 탭 URL 미반영
- 스튜디오 탭(소개/포스트/멤버십/클럽/커뮤니티)이 클라이언트 `<button>` 상태 → 새로고침 시 초기화, 딥링크/뒤로가기 불가. `?tab=` 쿼리 동기화 권장(발표 중 새로고침 대비).

### 11. 커스텀 404 페이지 없음
- 존재하지 않는 creator/program/post 접근 시 영어 기본 "404 This page could not be found". 한글 커스텀 `not-found.tsx` 권장.

### 12. 신청 실패 에러 카피
- 신청 검증 실패 시 API의 generic "Validation failed" 영문이 그대로 노출(P0 #1 해결 시 빈도 감소하나, 한글 메시지 매핑 권장).
