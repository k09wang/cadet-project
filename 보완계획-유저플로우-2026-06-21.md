## 작업 1 — 멤버십 플랜 관리 페이지 + 진입점 (유저플로우 n8)

- `dashboard/creator/memberships/new`(생성)만 존재. 플랜 **목록/수정/삭제 관리 페이지가 없음**.

2. **신규** `src/app/(app)/dashboard/creator/memberships/page.tsx`

   - 플랜 카드 목록 + "플랜 생성" 링크(`/dashboard/creator/memberships/new`)
   - 각 카드: 제목·가격·설명 + "수정" / "삭제" 액션
   - 프로필 미보유 시 안내 (대시보드 H3 패턴 재사용)

3. **신규** `src/app/(app)/dashboard/creator/memberships/[id]/edit/page.tsx`

4. **API 확장** `src/app/api/membership-plans/[id]/route.ts` **신규**
   - `PATCH`: 본인 플랜 검증 후 수정 (`membershipPlanUpdateSchema` — `validation/membership.ts`에 partial 스키마 추가)
   - `DELETE`: 본인 플랜 검증 후 삭제 (활성 `Membership` 존재 시 차단 정책 결정 — 아래 미결정 참고)

   - `route.test.ts` 추가

5. **대시보드 링크 추가** `src/app/(app)/dashboard/creator/page.tsx`
   - 하단 `grid grid-cols-2` 액션 섹션에 `멤버십 관리` → `/dashboard/creator/memberships` 링크 1개 추가

## 작업 2 — AI 가격·혜택 추천 위치 정합 (유저플로우 n8 → n10)

### 문제
- 유저플로우상 AI 추천은 **멤버십 플랜**(n8→n10)에 연결.
- 실제 구현(`AiSuggestPanel`, `/api/programs/ai-suggest`, `lib/ai/suggest.ts`)은 **프로그램 생성**에만 존재.
- 멤버십 생성 화면(`memberships/new`)에 AI 추천 없음.

### 방향 (확정: A안 — 멤버십 생성에도 AI 추천 추가)

### A안 실행 변경
1. `src/lib/ai/suggest.ts` 확인 후 멤버십용 추천 함수 분기
   - 멤버십은 "가격 + 혜택" 중심 (프로그램의 주차 구성 불필요)
   - `suggestMembership(input)` 추가 또는 기존 `suggestProgram` 입력 타입 확장
   - Mock 폴백 유지 (NFR-001: 예외 throw 금지)
2. **신규** `src/app/api/membership-plans/ai-suggest/route.ts`
   - `programs/ai-suggest/route.ts` 패턴 복제, 멤버십 입력 스키마 적용
3. 멤버십 생성 화면을 클라이언트 래핑
   - `NewProgramClient` ↔ `AiSuggestPanel` 패턴 참고하여 `NewMembershipClient` 작성
   - "AI 추천 받기" → 추천 결과를 `MembershipPlanForm`의 `priceKrw`/`description`에 주입(remount rev 패턴)
   - 단, 현재 `memberships/new`는 Server Action(`createPlanAction`) 사용 → 폼 주입 방식만 클라이언트로 전환, 제출은 기존 API 유지
4. 테스트 추가 (`NewMembershipClient.test.tsx`, `route.test.ts`)

### 완료 기준
- 멤버십 생성 화면에서 AI 추천 → 가격·혜택 폼 반영
- 기존 프로그램 AI 추천 회귀 없음

## 작업 3 — 커뮤니티 진입점 추가 (유저플로우 n40/n41)

### 현황
- 커뮤니티는 **스튜디오 공개 페이지(`creators/[creatorId]`)의 탭**으로 이미 구현됨 (소개/포스트/멤버십/클럽/커뮤니티 — `StudioTabs.tsx`).
- API(`/api/community-posts`), 컴포넌트(`CommunityPanel/Composer/List`), 쿼리(`listCommunityPosts`) 모두 존재.
- **빠진 것: 유저플로우상 "크리에이터 대시보드(n5) → 커뮤니티", "탐색 홈(n23) → 커뮤니티" 진입 동선.**

### 방향 (확정: A안 — 진입 동선만 보강)
- 대시보드/탐색홈에서 "커뮤니티" 안내·링크 추가
  - 단, 현 커뮤니티는 *크리에이터별*이므로 글로벌 단일 커뮤니티가 아님 → 링크 대상은 "내가 가입한 스튜디오 / 내 스튜디오의 커뮤니티 탭"으로 연결
  - fan home: `dashboard/fan` 둘러보기 섹션에 "커뮤니티" 카드 추가 → 가입 멤버십의 스튜디오 커뮤니티 탭으로 유도 (또는 `/creators` 안내)
  - creator dashboard: 액션 그리드에 "내 커뮤니티" → 본인 스튜디오 공개페이지 커뮤니티 탭(`/creators/{profileId}?tab=community`) 링크
  - `StudioTabs`가 `?tab=` 쿼리로 초기 탭 진입을 지원하는지 확인, 미지원이면 초기 탭 prop/쿼리 처리 추가
- **B안 (고비용): 글로벌 커뮤니티 허브 페이지 신설** — 유저플로우를 문자 그대로 해석. MVP 범위 초과 가능성, 별도 SPEC 권장.

### A안 실행 시 변경
1. `StudioTabs.tsx` — `?tab=community` 등 쿼리/초기 탭 진입 지원 (미지원 시)
2. `src/app/(app)/dashboard/creator/page.tsx` — "내 커뮤니티" 링크 추가
3. `src/app/(app)/dashboard/fan/page.tsx` — 둘러보기 섹션에 "커뮤니티" 진입 추가
4. 관련 테스트 갱신

### 완료 기준
- 대시보드/탐색홈에서 커뮤니티로 이어지는 클릭 동선 존재
- 기존 스튜디오 탭 동작 회귀 없음
